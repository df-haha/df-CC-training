Step 3：CLAUDE.md（續）

## 程式碼規範
- 元件用 PascalCase
- 函式用 camelCase
- 用 FC 定義 React 元件

## 資料結構
```typescript
interface Todo {
  id: string;
  title: string;
  completed: boolean;
  dueDate?: string;
  createdAt: string;
}
```

Q&A：程式碼不對怎麼辦？

解決方案

1. 給更具體的需求描述

2. 提供驗證測試案例

3. 讓 Claude 解釋它的程式碼

4. 分步執行，每步確認

大部分問題都是需求不夠清楚造成的

Q&A：Context 滿了怎麼辦？

解決方案

1. 用 /compact 壓縮對話

2. 用 /clear 清除不需要的對話

3. 只引用需要的檔案（@specific-file.ts）

4. 維持良好的 CLAUDE.md

Q&A：什麼時候用 Plan Mode？

適合用 Plan Mode 不需要 Plan Mode
新功能（多檔案異動） 簡單問答
重構 單檔案小修改
架構變更 Debug 單一問題
不確定怎麼做時 已經很清楚要做什麼

Q&A：MCP 和 Skill 怎麼選？

情境 推薦方案
涉及認證（密碼、Token） MCP
單純流程自動化 Skill
一次性腳本 Skill + Python
需要外部 API MCP
重複使用的工作流程 Skill

核心心法

四大核心心法

1. Claude Code = 工人，不只是顧問

它真的會執行，要小心權限

2. Context Engineering > Prompt Engineering

管理好 AI 的記憶比寫好問題更重要

3. SDD：先講清楚需求再動手

「先不要寫，幫我釐清需求」

4. 驗證是最重要的事

給 Claude 驗證方式，品質提升 2-3 倍
