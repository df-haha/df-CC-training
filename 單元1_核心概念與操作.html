1.1 什麼是終端機？

終端機 = 那個黑底白字的視窗

你可能聽過這些名詞：

終端機、Terminal、命令列、CLI、Command Line

它們都是同一個東西：
你打一行指令，電腦就執行一件事

常見指令範例

你打的指令 電腦做的事
dir 列出資料夾裡的檔案
mkdir photos 建立一個叫 photos 的資料夾
del temp.txt 刪除 temp.txt 這個檔案
code . 打開 VS Code

為什麼要知道這個？

關鍵概念

Claude Code 就是能在終端機下指令的 AI

這代表它真的能對你的電腦做事

不只是給建議，而是真的會執行

1.5 CLI 版簡介

CLI 版怎麼用？

在終端機直接輸入 claude 就可以啟動：

# 基本啟動
claude

# 帶問題啟動
claude "這個專案是做什麼的？"

# 用 Plan 模式
claude -p "分析這個功能的架構"

# 指定模型
claude --model opus "幫我重構這段程式碼"

VSCode vs CLI 功能差異

功能 CLI 版 VSCode 版
全部斜線指令 ✅ 全部支援 ⚠️ 部分支援
MCP Server 設定 ✅ 可設定 ❌ 需透過 CLI
Diff View 視覺化 ⚠️ 文字版 ✅ 圖形化比較
拖曳檔案 ❌ ✅ 支援
裝 B 指數 ✅ 高（黑底白字很專業） ⚠️ 普通

CLI 專屬進階功能

以下功能目前只有 CLI 版支援

/fork 指令（CLI 限定）

功能：分叉對話，嘗試不同做法

使用情境

想試試不同做法，但不確定哪個好

開啟一個新分支，保留目前所有對話內容

可以在分支裡嘗試 A 方案，原本的對話嘗試 B 方案

最後選一個比較好的繼續

/export 指令（CLI 限定）

功能：把對話匯出成 markdown 或 json 檔案

用途：

  • 分享給同事看
  • 備份重要對話
  • 寫文件時引用

恢復對話（CLI 限定）

Claude Code 會保留你的對話歷史

操作方式：

  • 輸入 claude --resume 恢復上一次對話
  • 可按關鍵字搜尋，或按時間瀏覽

/rewind 指令（CLI 限定）

功能：回溯對話，退回前幾步重來

你
/rewind
Claude
顯示對話歷史，讓你選要回到哪一步：
[1] 幫我加登入功能
[2] 改成用 JWT
[3] 加上 refresh token ← 從這裡開始出問題
[4] 改成存在 cookie

選擇 2，對話就回到步驟 2，後面的修改都取消

什麼時候用哪個？

情境 推薦版本
日常開發、需要看 Diff ✅ VSCode 版
遠端伺服器（SSH 連線） ✅ CLI 版
設定 MCP Server ✅ CLI 版
電腦很慢/記憶體不夠 ✅ CLI 版（輕量快速）
本課程以 VSCode 版為主，但對話歷史共享，可跨版本恢復

1.6 專案管理原則

錯誤做法：一個專案硬幹到底

我的超大專案/
├── 客戶A網站/
├── 客戶B App/
├── 內部工具/
└── 測試專案/
    └── CLAUDE.md  ← 一個 CLAUDE.md 管所有東西？
CLAUDE.md 會變得超長、不同專案的規則會衝突、Claude 容易搞混

正確做法：每個專案獨立

工作區/
├── 客戶A網站/
│   └── CLAUDE.md  ← 客戶 A 的技術棧、規範
├── 客戶B App/
│   └── CLAUDE.md  ← 客戶 B 的技術棧、規範
├── 內部工具/
│   └── CLAUDE.md  ← 內部規範
└── 測試專案/
    └── CLAUDE.md  ← 測試用設定

專案拆分四原則

原則 說明 範例
業務邊界 不同客戶/產品分開 客戶 A 和客戶 B 分開
技術棧 不同框架分開 React 和 Vue 專案分開
團隊分工 不同團隊負責的分開 前端和後端分開
生命週期 獨立部署的分開 微服務各自獨立

實作練習

練習 1：基本操作

  • 開啟 VS Code，打開任意檔案
  • 點擊右上角 ✱ 圖示開啟 Claude
  • 輸入：「列出這個資料夾有什麼檔案」

練習 2：檔案互動

  • 用 @ 提及一個檔案
  • 拖曳一個檔案到對話框
  • 選取程式碼 → 右鍵 → Ask Claude

練習 3：指令練習

  • 執行 /context 查看 Context 用量
  • 執行 /help 查看可用指令
