5.1 從重複輸入到 Skills

沒有 Skill 之前：每次都要複製貼上

你
幫我寫一個表單功能

工作規則：
1. 先用選擇題釐清需求，一次只問一個問題
2. 列出驗收標準，我確認後才開始
3. 做完要實際測試，不能說「應該可以」
4. 如果測試失敗要自己修正

每次都要複製貼上這些規則！太累了！

有了 Skill 之後：一個指令搞定

你
/reliable-dev 幫我寫一個表單功能
Claude
（自動套用規則）
好的，讓我先確認需求...
[一次問一個問題]
...
[確認驗收標準]
...
[實際測試並回報結果]
一個指令，省去每次重複說明！

Skills 定義

Skills = 可重複使用的指令範本

把你常用的規則、流程、格式打包成一個指令

以後只要打 /指令名稱 就能套用

就像手機的「快捷指令」或 Excel 的「巨集」

Skills 的價值

好處 說明
省時間 不用每次複製貼上規則
一致性 每次執行都遵守相同規則
不會忘記 規則存在檔案裡，不怕漏掉
可分享 團隊可以共用相同的 Skills

兩種 Skill 類型預告

Agent Skills

Claude 自己判斷什麼時候該用

語意理解，自動觸發

存放：.claude/skills/

VS

Custom Commands

你決定什麼時候要用

打 /指令 明確觸發

存放：.claude/commands/

5.2 Agent Skills vs Custom Commands

Agent Skills：彈性、智慧判斷

特色

Claude 會根據對話內容，自動判斷：

「現在討論的東西，符合哪個 Skill 的 description？」

符合的話，就自動啟用那個 Skill 的規則。

存放位置：.claude/skills/

Agent Skills 範例：翻譯專家

# .claude/skills/translator.md
---
name: translator
description: 當對話涉及翻譯、多語言、在地化時啟用
---

翻譯規則：
1. 保持原文的語氣和風格
2. 專有名詞維持原文，括號附上翻譯
3. 技術術語使用業界慣用翻譯
4. 如果不確定，列出多個選項讓使用者選

Agent Skills 觸發方式

你
這段英文幫我翻成中文...
Claude
（語意理解到「翻譯」相關）
（自動啟用 translator Skill）
（按照 Skill 定義的規則翻譯）

你沒有打任何指令，Claude 自己判斷需要這個技能！

Agent Skills 優缺點

優點 缺點
不用記指令，自動觸發 Claude 可能判斷錯誤
更彈性，語意理解 執行結果比較不可預測
適合「領域專業」類型的規則 該啟用沒啟用、不該啟用卻啟用

Custom Commands：穩定、可預測

特色

有時候你不想讓 Claude 自己判斷，

你就是要它「現在、立刻、馬上」執行某件事。

打 /review，Claude 就執行程式碼審查

每次都一樣，不會有意外。

存放位置：.claude/commands/

Custom Commands 範例：/review

# .claude/commands/review.md
---
name: review
description: 程式碼審查
---

請審查這段程式碼，檢查：
1. 邏輯正確性
2. 安全性
3. 可讀性

輸出格式：
- 優點：
- 問題：
- 建議：

Custom Commands 使用方式

你
/review @src/login.js
Claude
（不管現在在聊什麼，立刻執行審查）
（按照指令定義的格式輸出）

優點：...
問題：...
建議：...

Custom Commands 優缺點

優點 缺點
穩定執行，每次都一樣 指令會越來越多，記不住
你完全控制何時觸發 要手動打指令
不會誤判 不會自動啟用

核心差異

核心問題：誰決定「何時觸發」？

Agent Skills = Claude 決定（語意理解）→ 彈性

Custom Commands = 你決定（打 /指令）→ 穩定

.claude/
├── skills/      ← Claude 自動判斷觸發
│   ├── translator.md
│   └── react-expert.md
│
└── commands/    ← 你手動觸發（/指令）
    ├── review.md
    └── explain.md

什麼時候用哪個？決策表

你的需求 選擇 原因
希望 Claude 聰明判斷 Agent Skills 彈性、不用記指令
每次都要一模一樣執行 Custom Commands 穩定、可預測
領域專業規則 Agent Skills 遇到相關話題自動套用
固定格式的輸出 Custom Commands 格式一致
怕 Claude 判斷錯誤 Custom Commands 你來控制

實作：建立 Custom Command

Step 1：建立資料夾

# 在專案根目錄建立
mkdir -p .claude/commands

Step 2：建立指令檔案 .claude/commands/explain.md

---
name: explain
description: 解釋程式碼
---

請解釋這段程式碼，包含：
1. 這段程式碼在做什麼（一句話）
2. 執行流程
3. 使用範例

用初學者能理解的方式說明。

Step 3：使用 /explain @src/utils/helper.js

實作：建立 Agent Skill

Step 1：建立資料夾

# 在專案根目錄建立
mkdir -p .claude/skills

Step 2：建立技能檔案 .claude/skills/react-expert.md

---
name: react-expert
description: 當對話涉及 React、元件、hooks、狀態管理時啟用
---

React 開發規則：
1. 元件使用 Function Component + Hooks
2. 狀態管理優先用 useState/useReducer
3. 副作用用 useEffect，注意 cleanup
4. 命名用 PascalCase

Step 3：自動生效 — 當你討論 React 時會自動啟用

5.3 MCP 外部連接能力

Skills 的限制

Skills 只能處理「對話」和「檔案」

Skills 做不到：

- 連接資料庫查資料

- 發 Slack 訊息

- 操作瀏覽器

- 讀取 Notion 文件

這些需要「連接外部系統」的能力。

MCP 是什麼？

MCP = Model Context Protocol = Claude 的外掛系統

MCP 讓 Claude 能連接外部工具

就像手機裝 App：

- 裝了「銀行 App」→ 手機能查餘額

- 裝了「Slack MCP」→ Claude 能發訊息

- 裝了「資料庫 MCP」→ Claude 能查資料

常見 MCP 列表

MCP 能做什麼 使用場景
PostgreSQL 查詢資料庫 查數據、分析報表
Slack 發送/讀取訊息 團隊通知
Playwright 操作瀏覽器 網頁測試、截圖
Context7 查最新技術文件 查 React/Vue 文件
Todoist 任務管理 待辦清單

MCP 安裝與使用

安裝：

claude mcp add postgres     # 安裝資料庫 MCP
claude mcp add slack        # 安裝 Slack MCP
claude mcp list             # 查看已安裝的 MCP

使用：裝好之後，Claude 會自動知道什麼時候該用。

你
昨天有多少新用戶？
Claude
（自動使用 PostgreSQL MCP 查詢）

5.4 Skills vs MCP 選擇指南

兩者的差別

Skills 像「食譜」

告訴廚師做菜的步驟

定義「怎麼做」

VS

MCP 像「廚房設備」

烤箱、冰箱、攪拌機

提供「能做什麼」

兩者可以搭配使用

用「食譜」（Skill）指導，用「設備」（MCP）執行

MCP 優點

優點 說明
認證安全 密碼、Token 由 MCP Server 持有，不會出現在對話裡
能力封裝 可以限制只能做特定操作（如：只能查詢、不能刪除）
狀態持久 保持連線（如：瀏覽器不用每次重開）
裝好就用 別人寫好的，不用自己寫程式

MCP 缺點

每個 MCP 的工具描述都會佔 Context 空間！裝太多會擠壓對話空間。
缺點 說明
佔用 Context 每個 MCP 都會佔用 Context 空間
需要持續運行 MCP Server 要一直跑著，佔系統資源
設定麻煩 有些 MCP 需要設定 API Key、OAuth 等
功能受限 只能用 MCP 提供的功能

Skill + Python 優點

優點 說明
超級彈性 想做什麼都可以，不受 MCP 功能限制
不佔 Context 不會像 MCP 一直佔著空間
不需要額外設定 有 Python 就能跑
一次性任務友好 用完就刪，不留痕跡

Skill + Python 缺點

缺點 說明
認證風險 密碼可能出現在腳本裡，或要自己管環境變數
要會 Python 如果你不熟 Python，可能看不懂 Claude 寫的腳本
沒有能力限制 Python 什麼都能做，也可能做錯事
每次重來 沒有狀態持久，每次都要重新連線

全面比較表

考量面向 MCP Skill + Python
認證安全 更安全 要自己管理
Context 佔用 會一直佔用 不佔用
彈性 受限於 MCP 功能 想做什麼都行
上手難度 裝好就用 要會 Python
穩定性 Server 可能掛 只依賴 Python

決策流程

該用 MCP 還是 Skill + Python？

1. 這個任務涉及密碼、Token 嗎？

→ 是：優先考慮 MCP（更安全）

2. 這是經常要做的事，還是一次性的？

→ 經常做：考慮 MCP（方便）

→ 一次性：Skill + Python（不佔 Context）

3. 有現成的 MCP 嗎？

→ 有：用 MCP（省事）

→ 沒有：Skill + Python（自己寫）

整合範例：Skill + MCP

# .claude/commands/daily-report.md
---
name: daily-report
description: 每日報表
---

請執行以下步驟：

1. 用資料庫 MCP 查詢昨日數據：
   - 新用戶數
   - 訂單數
   - 營收

2. 整理成報表格式

3. 用 Slack MCP 發送到 #daily-report 頻道

Skill 定義流程，MCP 負責執行！

5.5 Plugin 與回顧

什麼是 Plugin？

Plugin = 社群開發的 MCP 擴充

MCP 是技術規格（像 USB 標準）

Plugin 是基於 MCP 做出來的實際工具（像各種 USB 裝置）

MCP = 插座標準

定義規格

=

Plugin = 各種電器

別人做好的，插上就能用

推薦 Plugin

Plugin 功能 使用場景
Context7 查最新技術文件 查 React/Vue/Node 最新語法
Playwright 操作瀏覽器 截圖、爬資料、測試
PostgreSQL 查詢資料庫 分析數據、產生報表
Todoist 任務管理 GTD 系統整合

Plugin 安裝指令

# 安裝 Plugin
claude mcp add context7
claude mcp add playwright
claude mcp add postgres

# 查看已安裝的 Plugin
claude mcp list

# 移除 Plugin
claude mcp remove context7
記得：只裝你真正會用到的 Plugin，不要貪多，以免佔用過多 Context！

推薦 Skills

Skill 功能 適合誰
ui-ux-pro-max 專業 UI/UX 設計 設計師、全端開發
frontend-design 前端設計產生 前端開發者
code-simplifier 簡化複雜程式碼 所有開發者
code-review 程式碼審查 所有開發者
ui-ux-pro-max 支援 50 種設計風格、21 種配色方案！
